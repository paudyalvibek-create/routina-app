# Accessibility Statement - Routina

## Commitment to Accessibility
Routina is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and applying the relevant accessibility standards.

## Conformance Status
Routina aims to conform to WCAG 2.2 Level AA standards.

## Accessibility Features

### Keyboard Navigation
- ✅ All interactive elements accessible via keyboard
- ✅ Logical tab order throughout the application
- ✅ Visible focus indicators on all focusable elements
- ✅ Skip navigation links available
- ✅ Keyboard shortcuts documented

### Screen Reader Support
- ✅ Semantic HTML5 elements used throughout
- ✅ ARIA labels on all interactive components
- ✅ Alt text for images (when applicable)
- ✅ Form labels properly associated
- ✅ Error messages announced to screen readers
- ✅ Dynamic content changes announced

### Visual Design
- ✅ Color contrast ratios meet WCAG AA standards (4.5:1 for normal text)
- ✅ Text resizable up to 200% without loss of functionality
- ✅ No information conveyed by color alone
- ✅ Focus indicators visible and clear
- ✅ Consistent navigation throughout

### Forms & Inputs
- ✅ All form fields have visible labels
- ✅ Required fields clearly marked
- ✅ Error messages are descriptive and helpful
- ✅ Error identification and suggestions provided
- ✅ Form validation accessible

### Content Structure
- ✅ Headings used in logical order (H1 → H2 → H3)
- ✅ Lists marked up properly
- ✅ Tables include headers (when applicable)
- ✅ Content flows logically without CSS
- ✅ Language of page declared

### Interactive Elements
- ✅ Buttons have descriptive labels
- ✅ Links have meaningful text
- ✅ Icons include text alternatives
- ✅ Modal dialogs trap focus appropriately
- ✅ Loading states communicated

### Responsive Design
- ✅ Mobile-first approach
- ✅ Touch targets meet minimum size (44x44px)
- ✅ Responsive across devices
- ✅ Content reflows without horizontal scrolling
- ✅ Zoom functionality preserved
